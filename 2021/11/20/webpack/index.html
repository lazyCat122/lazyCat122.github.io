    1. 运行 npm install webpack-dev-server -D命令，安装支持项目自动打包的工具

    2. 修改 package.json -> scripts 中的 dev 命令如下：

    3. 将 src -> index.html 中，script 脚本的引用路径，修改为 “/bundle.js” , / 表示根路径

    4. 运行 npm run dev命令，重新进行打包

    5. 在浏览器中访问 http://localhost:8080地址，查看自动打包效果

    注意：访问http://localhost:8080/bundle.js, 可以访问到bundle.js，说明 bundle.js在我们项目的根目录中，在项目目录中看不到它，是因为webpack-dev-server生成的bundle.js，并没有放到物理磁盘中，而是放到内存中，它是一个虚拟的看不见的文件(bundle.js)，所以script 脚本的引用路径，需要修改为 “/bundle.js”

    注意：1. webpack-dev-server 会启动一个实时打包的 http 服务器

    ​ 2. webpack-dev-server 打包生成的输出文件，默认放到项目的根目录中，而且是虚拟的，看不见的

    webpack 中加载器的基本使用

    webpack 中的加载器

    1. 在实际开发过程中，webpack 默认只能打包处理以.js后缀名结尾的模块，其他非 .js后缀名结尾的模块，webpack 默认处理不了，需要调用 loader加载器才可以正常打包，否则会报错

      loader 加载器可以协助 webpack 打包处理特定的文件模块，比如：

      • less-loader 可以打包处理 .less 相关的文件
      • sass-loader 可以打包处理 .scss 相关的文件
      • url-loader 可以打包处理 css 中与 url 路径相关的文件

    2. loader 的调用过程

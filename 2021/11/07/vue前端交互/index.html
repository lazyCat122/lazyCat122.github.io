    前后端交互：异步编程与Promise概述

    多次异步调用的依赖分析：

    • 多次异步调用的结果顺序不确定
    • 异步调用结果如果存在依赖，就需要嵌套

    异步调用结果如果存在依赖，就需要嵌套

    如果嵌套的层数比较多，就会变得很复杂。这被称为回调地狱，使用promise解决回调地狱

    promise

    promise是异步编程的一种解决方案，从语法上讲，promise是一个对象，从它可以获取异步操作的消息。

    promise优点：

    • 可以避免多层异步调用嵌套问题(回调地狱)
    • promise对象提供了简洁的API，使得控制异步操作更加容易

    promise常用API

    • p.then() 得到异步任务的正确结果
    • p.catch() 获取异常信息，专门用来处理rejected状态
    • p.finally() 成功与否都会执行

    对象方法

    • Promise.all() 并发处理多个异步任务，所有任务都执行完成才能得到结果
    • Promise.race() 并发处理多个异步任务，只要有一个任务完成就能得到结果

    接口调用-axios用法

    axios是一个第三方包，需要引用

    axios是一个基于Promise用于浏览器和nodejs的HTTP客户端

    特征：

    • 支持浏览器和nodejs
    • 支持promise
    • 能拦截请求和响应
    • 自动转换JSON数据

    axios 的基本用法

    axios 的常用API

    • get: 查询数据
    • post：添加数据
    • put：修改数据
    • delete：删除数据

    1. GET传递参数

      • 通过url传递参数
      • 通过params选项传递参数

      注意：在前端通过params选项传递参数，后端接收参数用req.query

    2. POST传递参数

      两种方式

      通过选项传递参数(默认传递的是json格式的数据)

      传递表单格式的数据，通过URLSearchParams传递参数（application/x-www-form-urlencoded）

    axios 的响应结果

    响应结果的主要属性：

    • data：实际响应回来的数据
    • headers：响应头信息
    • status：响应状态码
    • statusText：响应状态信息

    axios的全局配置

    • axios.defaults.timeout = 3000; // 超时时间
    • axios.defaults.baseURL = ‘http://localhost:3000/' // 默认地址
    • axios.defaults.headers[ ‘mytoken’ ] = ‘ajkaljf’ // 设置请求头

    接口调用-async/await用法

    async/await的基本用法

    • es7引用得你语法，更方便的进行异步操作
    • async关键字用于函数上（async函数的返回值是Promise实例对象）
    • await关键字用于async函数当中（await可以得到异步的结果）

    async/await处理多个异步请求

    注释：

    1. 如果await后面的是一个 Promise 对象，await 就忙起来了，它会阻塞后面的代码，等着 Promise 对象 resolve，然后得到 resolve 的值，作为 await 表达式的运算结果。
    2. 如果await后面的不是一个 Promise 对象，那 await 表达式的运算结果就是它等到的东西。

    组件的命名方式

    短横线方式：Vue.component('my-component', {} )

    驼峰方式：Vue.component('MyComponent', {} )

    全局组件，不仅可以用到根组件，也可以用到其它的组件

    这是规则：注意：如果使用驼峰式命名组件，那么在使用组件的时候，只能在字符串模板中用驼峰的方式使用组件，但是在普通的标签模板中，必须使用短横线的方式使用组件。因为DOM元素不区分大小写

    组件间数据交互

    父组件向子组件传值

    1. 组件内部通过props接收传递过来的值

    2. 父组件通过属性将值传递给子组件

    vue实例是一个根组件

    1. props属性名规则

      • 在props中使用驼峰形式，模板中需要使用短横线的形式
      • 字符串形式的模板中没有这个限制

    2. props属性值类型

      • 字符串String
      • 数值Number
      • 布尔值Boolean
      • 数组Array
      • 对象Object

    子组件向父组件传值

    1. 子组件通过自定义事件向父组件传递信息

      <button v-on:click='$emit("enlarge-text")'>扩大字体</button>

    2. 父组件监听子组件的事件

      <menu-item v-on:enlarge-text='fontSize += 0.1'></menu-item>

    3. 子组件通过自定义事件向父组件传递信息

      <button v-on:click='$emit("enlarge-text", 0.1)'>扩大字体</button>

      0.1是要传递给父组件的参数

    4. 父组件监听子组件的事件

      <menu-item v-on:enlarge-text='fontSize += $event'></menu-item>

      $event代表接受的子组件的参数

    非父子组件间传值

    1. 单独的事件中心管理组件间的通信

      var eventHub = new Vue()

    2. 监听事件与销毁事件

      eventHub.$on('add-todo', addTodo)

      eventHub.$off('add-todo')

    3. 触发事件

      eventHub.$emit('add-todo', id)

    组件插槽

    具名插槽

    根据名称进行匹配，没有名称的放到默认插槽中

    作用域插槽

    应用场景：父组件对子组件的内容进行加工处理

    流程：就是在父组件中可以获取子组件的数据，并且对这些数据进行加工和处理。获取数据的方式就是通过slot-scope这个属性来得到，它当中得到的值就是从子组件中slot的属性中绑定的这个info
